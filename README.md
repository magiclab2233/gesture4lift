# 无接触智慧电梯控制器

基于 **MediaPipe HandLandmarker** 的实时手势识别系统，实现零接触控制电梯楼层，适用于医院、公共电梯间等对卫生要求较高的场景。

---

## 核心创意

针对公共场所卫生需求，设计一套通过特定手势（数字手势 1-5、五个指尖并拢确认）控制电梯楼层的系统。无需触摸实体按键，降低交叉感染风险。

### 手势说明

- **数字 1-5**：选择目标楼层（1 楼 至 5 楼）。
- **五个指尖并拢确认**：手掌面向摄像头，将五个指尖收拢靠近，确认当前选择。

> 只有**手心朝向摄像头**时，系统才会响应手势，避免误触发。

## 技术方案与优化点

### 1. 指尖状态判定优化

原始方案仅通过比较指尖与关节的 `$y$` 坐标判断手指是否伸出，这在手掌倾斜或旋转时极不稳定。

**优化方案**：
- 采用**相对距离法**：比较 `指尖到手腕的距离` 与 `指根到手腕的距离`。
- 当比值超过 `config.FINGER_EXTEND_RATIO`（默认 1.15）时，判定为伸出。
- **大拇指严格判定**（关键优化）：
  - 采用**单一可靠策略**：比较拇指指尖到食指根的距离 vs 拇指根到食指根的距离。
  - 阈值通过分析 **401 条真实采集数据** 校准为 **0.75**（数据范围：标签 4 为 0.44~0.53，标签 5 为 0.91~0.97），彻底避免数字 **4**（拇指弯曲）被误识别为 **5**。
  - 当四指全部伸出时，只有拇指**明显张开**（远离食指根）才会判定为 5。

**优势**：对手掌旋转、远近缩放具有更强的鲁棒性，数字 5 的识别率大幅提升。

### 2. 手掌朝向过滤

通过向量叉积计算手掌平面法向量，仅在手心朝向摄像头时才允许触发控制指令（阈值 **0.014**），避免手背误识别导致误操作。该阈值同样经过真实数据验证，兼顾了并拢手势时的检测灵敏度。

### 3. 动态触发优化

**优化方案**：
- **五个指尖并拢触发**：采用双特征联合判定：
  1. **拇指靠近其他指尖**：拇指尖到最近的其他四指指尖距离 < 手掌宽度的 65%（排除数字4/5）
  2. **四指离散度小**：四指指尖到重心的平均距离 < 手掌宽度的 30%（排除数字3）
  该组合在真实采集数据上达到 **100% 准确率**，完全避免数字手势误触。
- **上升沿检测 + 冷却机制**：仅在并拢状态从“松开”变为“并拢”的瞬间触发一次，并设置 `CLICK_COOLDOWN_FRAMES`（默认 15 帧）防止连发。

### 4. 手势防抖（Buffer 机制）

手部自然抖动会导致单帧识别结果跳变。系统引入**环形缓冲区**，只有连续 `GESTURE_BUFFER_SIZE`（默认 8 帧）识别到同一数字手势时，才将该数字确认为“目标”。

**并拢防误识别优化**：
- 当检测到五个指尖并拢时，本帧的数字识别结果被设为 **-1（无效）**，**不会进入防抖缓冲区**。
- 这样可避免并拢手势被误识别为数字 **5**。
- 并拢期间，系统**保持上次已确认的数字**不变，确保用户可以先选好楼层，再放心并拢确认。
- 若缓冲区中出现不一致的数字（如从 3 切换到 5 的过程），缓冲区会被**自动清空**，防止过渡帧产生误触。

### 5. 中文显示优化

OpenCV 原生 `cv2.putText` 不支持中文，会导致状态提示和事件消息乱码。

**解决方案**：在 `ui_renderer.py` 中集成 **PIL (Pillow)** 绘制文字：
- 自动检测系统中文字体（优先微软雅黑，兜底黑体/宋体）。
- 所有 HUD 文本（状态栏、手势说明、事件提示、操作提示）均通过 PIL 渲染，**完美支持中英文混排**。
- 字体大小、颜色与 OpenCV 绘制风格保持一致，界面清晰无锯齿。

### 6. 电梯移动动画

为提升交互的真实感，系统在画面右侧绘制了**电梯井道与轿厢实时动画**：

- **井道结构**：垂直分布的 5 个楼层（1F ~ 5F），灰色外框 + 白色楼层标记线。
- **轿厢平滑移动**：确认楼层后，轿厢从当前楼层以 **ease-in-out 缓动曲线** 平滑滑向目标楼层，约 0.8 秒/层。
- **颜色状态反馈**：
  - **橙色**：电梯正在移动中。
  - **绿色**：到达目标楼层并停驻（高亮 1.5 秒后恢复）。
- **方向提示**：移动时在井道上方显示 `UP -> 5F` 或 `DOWN -> 2F` 的箭头提示。
- **动画期间锁定输入**：电梯移动过程中不再响应新手势，防止指令冲突。

### 7. 可视化 HUD 界面

OpenCV 实时叠加：
- 顶部状态栏（当前楼层 / 移动中提示）
- 左侧手势指南
- 右侧电梯井道动画
- 手部骨架与高亮关键点
- 大数字动态反馈（在手部上方实时显示识别到的数字）
- 右下角事件提示（触发结果）

## 测试与问题排查

本节记录开发过程中遇到的典型问题、测试方法与解决方案，供复现或二次开发时参考。

### 1. MediaPipe API 兼容性

**问题：** 参考网络教程使用 `mediapipe.solutions.hands` 时提示 `ModuleNotFoundError`。  
**原因：** MediaPipe 0.10.33 的 wheel 未打包 `solutions` 子模块。  
**解决：** 迁移至 **Tasks API** (`mediapipe.tasks.python.vision.HandLandmarker`)，并重写骨架绘制逻辑。

### 2. 模型文件路径含中文

**问题：** `hand_landmarker.task` 放在含中文路径下（如"桌面"），加载时报 `FileNotFoundError`。  
**原因：** MediaPipe 底层 C++ 扩展对 UTF-8 路径处理存在兼容性问题。  
**解决：** 若检测到路径含非 ASCII 字符，自动将模型复制到 `tempfile.gettempdir()` 后再加载。

### 3. 中文显示乱码

**问题：** `cv2.putText` 渲染中文时显示为乱码方块。  
**原因：** OpenCV 原生 `putText` 仅支持 ASCII。  
**解决：** 使用 **PIL (Pillow)** 的 `ImageDraw.text()` 叠加中文，再转回 OpenCV 帧显示。

### 4. 数字 5 被误识别为 4

**问题：** 伸出五指（数字 5）时频繁被识别为 4。  
**排查：** 最初采用"tip-to-wrist"和"tip-to-index-mcp"双策略判定大拇指，但手掌远近变化会干扰 tip-to-wrist。  
**数据验证：** 采集 101 条标签 4 和 51 条标签 5 的数据后发现：
- 标签 4 的 `tip_to_index_mcp / mcp_to_index_mcp` 比值：**0.44 ~ 0.53**
- 标签 5 的同一比值：**0.91 ~ 0.97**

两组数据无重叠。  
**解决：** 弃用双策略，仅保留 `tip_to_index_mcp / mcp_to_index_mcp` 单一策略，阈值通过数据校准定为 **0.75**。

### 5. 半弯曲手指被误判为伸出

**问题：** 手指半弯曲时仍被判定为"伸出"。  
**原因：** 此时 `tip_to_wrist` 可能仍大于 `mcp_to_wrist * 1.15`，但 `pip_to_wrist` 已大于 `tip_to_wrist`，说明指尖并未充分伸展。  
**解决：** 增加二次约束 `tip_to_wrist > pip_to_wrist * 1.05`，过滤半弯曲状态。

### 6. 确认动作的三次迭代

确认动作经历了完整的"提出 → 测试 → 否定 → 再提出"过程：

| 代次 | 方案 | 测试结果 | 结论 |
|------|------|---------|------|
| 第一代 | 食指前伸"Z 轴点击" | Z 坐标抖动剧烈 | ❌ 精度不足 |
| 第二代 | 食指-拇指捏合（OK） | 与数字 5 的拇指判定冲突 | ❌ 逻辑冲突 |
| 第三代 | **五指指尖并拢** | 与所有数字手势差异明显 | ✅ **最终采用** |

### 7. "五指并拢"与数字 4 混淆

**问题：** 数字 4 的五指尖最大跨度（1.00 ~ 1.08）与并拢动作（0.40 ~ 1.05）重叠，单一阈值无法区分。  
**深入分析：**
- 数字 4 的**拇指远离四指**，而并拢的**拇指紧贴四指**；
- 数字 3 的**四指分散**，而并拢的**四指紧密聚拢**。

**解决：** 提出**双特征联合判定**：
1. 拇指靠近度：拇指尖到最近的其他四指尖 < 手掌宽度 × 0.65
2. 四指离散度：四指指尖最大跨度 < 手掌宽度 × 0.30

在 401 条数据上验证：**准确率 100%**（数字 1~5 误触发 0 次，together 正确触发 96/96）。

### 8. 手掌朝向过滤过严

**问题：** 并拢手指确认时，系统偶尔提示"手背朝向，已忽略"。  
**原因：** 五指并拢时手指聚集，cross_z 变小，原阈值 `0.02` 过于严格。  
**解决：** 逐步下调至 **0.014**，兼顾并拢手势的检测灵敏度。

### 9. 手势缓冲区污染

**问题：** 先选数字 3 再并拢确认，过程中楼层发生跳变。  
**原因：**
1. 并拢帧被标记为 `-1`，进入缓冲区后稀释了已确认的数字；
2. 过渡帧可能被识别为其他数字，混入缓冲区。

**解决：**
1. `-1` 无效帧**不写入**环形缓冲区，保护已确认数字；
2. 缓冲区中出现不一致数字时**自动清空**，防止过渡帧误触。

## 数据采集与实验

本项目包含自主开发的数据采集工具 `data_collector.py`，用于采集手部关键点坐标并进行统计分析。

**使用方法**：
```powershell
.\venv\Scripts\python.exe data_collector.py
```
- 按键 `0~5`：标注当前手势为对应数字
- 按键 `T`：标注为"五指并拢"确认动作
- 按键 `N`：标注为"无手势"
- 按键 `S`：切换是否保存图像
- 按键 `Q`：退出

采集的数据保存在 `dataset/labels.csv` 中。基于 **401 条真实样本** 的统计分析，所有关键阈值均实现了数据驱动校准，确认动作检测准确率达到 **100%**。详见 `project_report.md` 第四章。
